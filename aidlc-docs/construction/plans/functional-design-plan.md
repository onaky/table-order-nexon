## 생성 실행 계획

### Step 1: 도메인 엔티티 상세 설계
- [ ] 모든 Entity의 필드, 타입, 제약조건 정의
- [ ] Entity 간 관계 (1:N, N:M) 상세 정의
- [ ] 인덱스 및 유니크 제약조건 정의

### Step 2: 비즈니스 로직 모델
- [ ] 주문 생성 플로우 상세 (세션 확인 → 주문 생성 → SSE 발행)
- [ ] 테이블 세션 라이프사이클 (시작 → 주문 → 이용 완료)
- [ ] 인증 플로우 (테이블 로그인, 관리자 로그인, 토큰 검증)
- [ ] 메뉴/재료 관리 플로우
- [ ] SSE 이벤트 발행 규칙

### Step 3: 비즈니스 규칙 정의
- [ ] 주문 관련 규칙 (상태 전이, 삭제 조건, 금액 계산)
- [ ] 인증 관련 규칙 (세션 만료, 로그인 시도 제한)
- [ ] 메뉴 관련 규칙 (필수 필드, 가격 범위, 순서)
- [ ] 테이블 관련 규칙 (세션 종료 조건, 이력 이동)

### Step 4: 프론트엔드 컴포넌트 상세
- [ ] 주요 페이지별 상태 관리 및 데이터 플로우
- [ ] 3D 뷰어 구현 전략
- [ ] 경과 시간 UI 로직
- [ ] 드래그 앤 드롭 상호작용 설계
