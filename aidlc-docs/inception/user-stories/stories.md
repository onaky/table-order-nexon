# User Stories

## 범례
- **형식**: As a [persona], I want [goal], So that [benefit]
- **우선순위**: MoSCoW (Must / Should / Could / Won't)
- **레이어**: 🔵 기본 기능 / 🟣 차별화 기능

---

# 🔵 기본 기능 레이어 (Core)

---

## Feature: 인증 및 세션 관리

### US-01: 테이블 태블릿 자동 로그인
**Priority**: Must

**As a** 고객 (김민수),
**I want** 테이블에 앉으면 별도 로그인 없이 바로 주문 화면을 볼 수 있기를,
**So that** 대기 시간 없이 즉시 메뉴를 탐색하고 주문할 수 있다.

**Acceptance Criteria**:
1. 관리자가 1회 초기 설정(매장 ID, 테이블 번호, 비밀번호) 완료 후 자동 로그인 활성화
2. 브라우저 새로고침 시에도 로그인 상태 유지
3. 로그인 실패 시 초기 설정 화면으로 이동
4. 저장된 인증 정보로 서버 인증 자동 수행

---

### US-02: 관리자 매장 인증
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 매장 식별자와 계정 정보로 로그인하여 관리 시스템에 접근하기를,
**So that** 내 매장의 주문과 메뉴를 안전하게 관리할 수 있다.

**Acceptance Criteria**:
1. 매장 식별자 + 사용자명 + 비밀번호로 로그인
2. 로그인 성공 시 JWT 토큰 발급 및 16시간 세션 유지
3. 브라우저 새로고침 시 세션 유지
4. 16시간 후 자동 로그아웃 및 재로그인 요구
5. 잘못된 로그인 시도 제한 (연속 실패 시 일시 차단)

---

## Feature: 메뉴 조회 및 탐색

### US-03: 카테고리별 메뉴 탐색
**Priority**: Must

**As a** 고객 (김민수),
**I want** 카테고리별로 분류된 메뉴를 쉽게 탐색하기를,
**So that** 원하는 종류의 음식을 빠르게 찾을 수 있다.

**Acceptance Criteria**:
1. 메뉴 화면이 기본 화면으로 항상 표시
2. 카테고리 탭/버튼으로 빠른 이동 가능
3. 각 메뉴 카드에 메뉴명, 가격, 설명, 이미지 표시
4. 터치 친화적 UI (최소 44x44px 터치 영역)
5. 카테고리 간 부드러운 전환

---

### US-04: 메뉴 상세 정보 확인
**Priority**: Must

**As a** 고객 (김민수),
**I want** 메뉴를 탭하면 상세 정보를 확인하기를,
**So that** 주문 전에 메뉴에 대해 충분히 파악할 수 있다.

**Acceptance Criteria**:
1. 메뉴 카드 탭 시 상세 페이지로 이동
2. 메뉴명, 가격, 상세 설명, 이미지 표시
3. 장바구니 추가 버튼 제공
4. 이전 화면으로 돌아가기 가능

---

## Feature: 장바구니

### US-05: 장바구니에 메뉴 추가
**Priority**: Must

**As a** 고객 (김민수),
**I want** 원하는 메뉴를 장바구니에 담기를,
**So that** 여러 메뉴를 한 번에 주문할 수 있다.

**Acceptance Criteria**:
1. 메뉴 상세 또는 카드에서 장바구니 추가 가능
2. 추가 시 시각적 피드백 (애니메이션/토스트)
3. 장바구니 아이콘에 담긴 수량 표시
4. 동일 메뉴 추가 시 수량 증가

---

### US-06: 장바구니 관리
**Priority**: Must

**As a** 고객 (김민수),
**I want** 장바구니에서 수량을 조절하거나 메뉴를 삭제하기를,
**So that** 주문 전에 원하는 대로 수정할 수 있다.

**Acceptance Criteria**:
1. 수량 증가/감소 버튼 제공
2. 메뉴 삭제 기능
3. 총 금액 실시간 계산 및 표시
4. 장바구니 비우기 기능
5. 페이지 새로고침 시에도 장바구니 유지 (localStorage)

---

## Feature: 주문

### US-07: 주문 생성
**Priority**: Must

**As a** 고객 (김민수),
**I want** 장바구니의 메뉴를 확인하고 주문을 확정하기를,
**So that** 선택한 음식이 주방에 전달된다.

**Acceptance Criteria**:
1. 주문 확정 전 최종 내역 확인 화면 표시
2. 주문 확정 버튼 클릭 시 서버에 주문 전송
3. 주문 성공 시: 주문 번호 5초 표시 → 장바구니 비우기 → 메뉴 화면 리다이렉트
4. 주문 실패 시: 에러 메시지 표시 + 장바구니 유지
5. 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

---

### US-08: 주문 내역 조회
**Priority**: Must

**As a** 고객 (김민수),
**I want** 현재 테이블에서 주문한 내역을 확인하기를,
**So that** 무엇을 주문했는지, 현재 상태가 어떤지 알 수 있다.

**Acceptance Criteria**:
1. 현재 테이블 세션의 주문만 표시 (이전 세션 제외)
2. 주문 시간 순 정렬
3. 각 주문: 주문 번호, 시각, 메뉴/수량, 금액, 상태 표시
4. 주문 상태: 대기중 / 준비중 / 완료

---

## Feature: 실시간 주문 모니터링 (관리자)

### US-09: 실시간 주문 대시보드
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 들어오는 주문을 실시간으로 대시보드에서 확인하기를,
**So that** 주문을 빠르게 처리할 수 있다.

**Acceptance Criteria**:
1. SSE 기반 실시간 업데이트 (2초 이내 표시)
2. 테이블별 카드 그리드 레이아웃
3. 각 카드: 테이블 번호, 총 주문액, 최신 주문 미리보기
4. 신규 주문 시 시각적 강조 (색상 변경, 애니메이션)
5. 주문 카드 클릭 시 전체 메뉴 목록 상세 보기

---

### US-10: 주문 상태 변경
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 주문 상태를 대기중 → 준비중 → 완료로 변경하기를,
**So that** 고객에게 현재 진행 상황을 알릴 수 있다.

**Acceptance Criteria**:
1. 주문 카드에서 상태 변경 버튼/드롭다운 제공
2. 상태 변경 즉시 반영 (SSE로 고객 화면에도 업데이트)
3. 상태 변경 시 시각적 피드백
4. 상태 흐름: 대기중 → 준비중 → 완료 (역방향 불가)

---

## Feature: 테이블 관리 (관리자)

### US-11: 테이블 초기 설정
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 테이블 번호와 비밀번호를 설정하기를,
**So that** 해당 테이블의 태블릿이 자동 로그인되어 고객이 바로 사용할 수 있다.

**Acceptance Criteria**:
1. 테이블 번호 및 비밀번호 입력/설정
2. 설정 완료 시 해당 테이블 세션 생성
3. 설정 정보 저장 및 자동 로그인 활성화
4. 성공/실패 피드백 표시

---

### US-12: 주문 삭제 (직권 수정)
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 잘못된 주문을 삭제하기를,
**So that** 고객의 주문 실수나 시스템 오류를 바로잡을 수 있다.

**Acceptance Criteria**:
1. 주문 삭제 버튼 제공
2. 확인 팝업 표시 (실수 방지)
3. 삭제 시 즉시 반영 및 테이블 총 주문액 재계산
4. 성공/실패 피드백 표시

---

### US-13: 테이블 이용 완료 (세션 종료)
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 고객이 떠난 후 테이블 이용 완료 처리를 하기를,
**So that** 다음 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다.

**Acceptance Criteria**:
1. 이용 완료 버튼 + 확인 팝업
2. 현재 세션 주문 내역 → 과거 이력으로 이동
3. 테이블 현재 주문 목록 및 총 주문액 0으로 리셋
4. 새 고객의 첫 주문 시 새 세션 자동 시작
5. 성공/실패 피드백 표시

---

### US-14: 과거 주문 내역 조회
**Priority**: Should

**As a** 관리자 (박준호),
**I want** 테이블별 과거 주문 내역을 조회하기를,
**So that** 이전 매출과 주문 패턴을 확인할 수 있다.

**Acceptance Criteria**:
1. "과거 내역" 버튼으로 접근
2. 테이블별 과거 주문 목록 (시간 역순)
3. 각 주문: 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
4. 날짜 필터링 기능
5. "닫기" 버튼으로 대시보드 복귀

---

## Feature: 메뉴 관리 (관리자)

### US-15: 메뉴 CRUD
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 메뉴를 등록, 수정, 삭제하기를,
**So that** 매장의 메뉴를 최신 상태로 유지할 수 있다.

**Acceptance Criteria**:
1. 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 업로드
2. 메뉴 수정: 모든 필드 수정 가능
3. 메뉴 삭제: 확인 팝업 후 삭제
4. 필수 필드 검증 (메뉴명, 가격, 카테고리)
5. 가격 범위 검증 (0 이상)
6. 카테고리별 메뉴 조회

# 🟣 차별화 기능 레이어 (Differentiation)

---

## Feature: 3D 메뉴 시각화 & 재료 정보

### US-16: 3D 메뉴 뷰
**Priority**: Should

**As a** 고객 (김민수),
**I want** 메뉴 상세 페이지에서 음식을 3D로 회전하며 확인하기를,
**So that** 실제 음식의 모습을 다양한 각도에서 미리 볼 수 있다.

**Acceptance Criteria**:
1. 블랙 배경의 메뉴 상세 페이지 중앙에 3D 모델 배치
2. 마우스 드래그 / 터치 스와이프로 360도 회전 가능
3. 60fps 이상의 부드러운 렌더링
4. 3D 모델 로딩 중 스켈레톤/로딩 표시
5. 3D 미지원 환경에서는 정적 이미지 폴백

---

### US-17: 재료 정보 확인
**Priority**: Should

**As a** 건강 관리 고객 (이지은),
**I want** 메뉴의 재료별 상세 정보(이미지, 칼로리, 맛 프로필)를 확인하기를,
**So that** 내 식단과 알레르기에 맞는 메뉴를 안전하게 선택할 수 있다.

**Acceptance Criteria**:
1. 메뉴 상세 페이지 우측에 재료 목록 세로 나열
2. 각 재료: 개별 이미지, 칼로리, 맛 프로필(매운맛/단맛 등) 표시
3. 재료 정보가 직관적으로 한눈에 파악 가능
4. 비건/채식 관련 재료 시각적 구분 (아이콘 또는 색상)
5. 재료 정보 없는 메뉴는 "정보 준비중" 표시

---

### US-18: 재료 관리 (관리자)
**Priority**: Should

**As a** 관리자 (박준호),
**I want** 메뉴별 재료 정보를 등록하고 관리하기를,
**So that** 고객에게 정확한 재료 정보를 제공할 수 있다.

**Acceptance Criteria**:
1. 재료 등록: 재료명, 이미지 업로드, 칼로리, 맛 프로필
2. 메뉴별 재료 연결/해제 (다대다 관계)
3. 재료 수정 및 삭제
4. 재료 이미지 파일 업로드 (서버 저장)
5. 재료 변경 시 연결된 메뉴에 즉시 반영

---

## Feature: 주문 상태 실시간 애니메이션

### US-19: 주문 상태 시각적 플로우 (고객)
**Priority**: Should

**As a** 고객 (김민수),
**I want** 내 주문 상태가 시각적 애니메이션으로 표시되기를,
**So that** 주문이 어떤 단계에 있는지 직관적으로 알 수 있다.

**Acceptance Criteria**:
1. 주문 내역에서 상태별 시각적 구분 (아이콘 + 색상 + 애니메이션)
2. 대기중 → 준비중 → 완료 단계별 프로그레스 표시
3. 상태 변경 시 부드러운 전환 애니메이션
4. 완료 상태에서 체크 애니메이션

---

## Feature: 주문 경과 시간 시각화 (관리자)

### US-20: 주문 경과 시간 색상 단계 표시
**Priority**: Must

**As a** 관리자 (박준호),
**I want** 각 주문이 들어온 후 경과 시간을 색상으로 직관적으로 확인하기를,
**So that** 오래된 주문을 놓치지 않고 우선 처리할 수 있다.

**Acceptance Criteria**:
1. 주문 카드에 실시간 경과 시간 카운터 표시 ("3분 전", "12분 전")
2. 0~5분: 초록색 (정상)
3. 5~10분: 노란색/주황색 (주의)
4. 10분 이상: 빨간색 (경고)
5. 경고 단계에서 펄스/글로우 애니메이션으로 시각적 긴급도 표현

---

## Feature: 매출 통계 대시보드

### US-21: 테이블별 실시간 매출 차트
**Priority**: Should

**As a** 관리자 (박준호),
**I want** 테이블별 매출 현황을 차트로 확인하기를,
**So that** 매장 운영 상황을 한눈에 파악하고 의사결정에 활용할 수 있다.

**Acceptance Criteria**:
1. 테이블별 매출 바 차트 또는 도넛 차트 표시
2. 실시간 데이터 반영 (주문 생성/삭제 시 업데이트)
3. 오늘 총 매출 요약 표시
4. 차트 영역 토글 (접기/펼치기) 가능

---

## Feature: 드래그 앤 드롭 메뉴 순서 관리

### US-22: 메뉴 순서 드래그 앤 드롭
**Priority**: Should

**As a** 관리자 (박준호),
**I want** 메뉴의 노출 순서를 드래그 앤 드롭으로 조정하기를,
**So that** 인기 메뉴나 추천 메뉴를 상단에 쉽게 배치할 수 있다.

**Acceptance Criteria**:
1. 메뉴 관리 화면에서 드래그 핸들 표시
2. 드래그 앤 드롭으로 순서 변경
