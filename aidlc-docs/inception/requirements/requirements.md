## 1. 프로젝트 개요

### 1.1 서비스 비전
디지털 주문 시스템을 통해 고객에게는 **프리미엄 시각 경험과 편리한 주문**을, 매장 운영자에게는 **효율적인 실시간 운영 환경**을 제공하는 차세대 테이블오더 플랫폼.

### 1.2 차별화 전략
- **3D 메뉴 시각화**: 메뉴 상세 페이지에서 음식을 3D로 회전하며 확인 가능
- **재료 기반 정보 제공**: 각 메뉴의 재료별 칼로리, 맛, 상세 정보 및 개별 이미지 제공
- **다크 모드 프리미엄 UI**: 블랙 배경 기반의 고급스러운 인터페이스
- **실시간 주문 상태 애니메이션**: 주문 → 준비중 → 완료 시각적 플로우
- **매출 통계 대시보드**: 테이블별 실시간 매출 차트
- **드래그 앤 드롭 메뉴 관리**: 직관적인 메뉴 순서 조정

## 2. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| **Frontend** | React + TypeScript | Vite 빌드 도구 |
| **Styling** | Tailwind CSS | 다크 모드 프리미엄 테마 |
| **3D Rendering** | React Three Fiber (Three.js) | 메뉴 3D 시각화 |

| **Charts** | Recharts / Chart.js | 매출 통계 |
| **DnD** | dnd-kit | 드래그 앤 드롭 메뉴 관리 |
| **Backend** | Node.js + Express + TypeScript | REST API |
| **Database** | MySQL | 관계형 데이터 |
| **ORM** | Prisma / TypeORM | DB 접근 계층 |
| **Realtime** | Server-Sent Events (SSE) | 실시간 주문 업데이트 |
| **File Upload** | Multer | 이미지 업로드 |
| **Auth** | JWT | 관리자 인증 |

## 4. 기능 요구사항

### 4.1 고객용 기능 (Customer Interface)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 (매장 ID, 테이블 번호, 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 세션 유지 (브라우저 새로고침 시에도)

#### FR-C02: 메뉴 조회 및 탐색
- 카테고리별 메뉴 분류 및 표시
- 메뉴 카드: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동 (탭/스크롤)
- 터치 친화적 UI (최소 44x44px 터치 영역)

#### FR-C03: 메뉴 상세 페이지 (차별화 핵심)
- **블랙 배경** 기반 프리미엄 레이아웃
- **중앙**: 메뉴 3D 모델 (마우스/터치로 회전 가능)
- **우측**: 재료 목록 세로 나열
  - 각 재료별 개별 이미지
  - 칼로리 정보
  - 맛 프로필 (매운맛, 단맛 등)
  - 알레르기 정보 (선택)
- 비건/건강 관리 사용자를 위한 직관적 정보 제공
- 장바구니 추가 버튼

#### FR-C04: 장바구니 관리
- 메뉴 추가/삭제
- 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- 로컬 저장 (localStorage, 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C05: 주문 생성
- 주문 내역 최종 확인 화면
- 주문 확정 버튼
- 주문 성공 시:
  - 주문 번호 표시 (5초)
  - 장바구니 자동 비우기
  - 메뉴 화면으로 자동 리다이렉트
- 주문 실패 시: 에러 메시지 + 장바구니 유지

#### FR-C06: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태
- **주문 상태 실시간 애니메이션** (대기중 → 준비중 → 완료)

### 4.2 관리자용 기능 (Admin Interface)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반 인증
- 16시간 세션 유지
- bcrypt 비밀번호 해싱
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링 대시보드
- **SSE 기반 실시간 업데이트** (2초 이내)
- 테이블별 카드 그리드 레이아웃
- 각 테이블 카드:
  - 테이블 번호
  - 총 주문액
  - 최신 주문 미리보기
- 주문 카드 클릭 시 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- **신규 주문 시각적 강조** (색상 변경, 애니메이션)
- **테이블별 실시간 매출 차트** (차별화)
- **주문 경과 시간 시각적 표시** (차별화):
  - 주문 카드에 경과 시간 실시간 카운터 표시 (예: "3분 전", "12분 전")
  - 시간 경과에 따른 색상 단계 변화 (정상 → 주의 → 경고):
    - 0~5분: 초록색 (정상)
    - 5~10분: 노란색/주황색 (주의)
    - 10분 이상: 빨간색 (경고, 펄스 애니메이션)
  - 오래된 주문일수록 시각적 긴급도 증가 (카드 테두리 강조, 글로우 효과)
  - 주문을 놓치지 않도록 경고 단계에서 카드 진동/펄스 애니메이션 적용

#### FR-A03: 테이블 관리
- 테이블 초기 설정 (번호, 비밀번호)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총액 재계산)
- 테이블 세션 종료 (이용 완료):
  - 확인 팝업
  - 주문 내역 → 과거 이력 이동
  - 현재 주문/총액 리셋
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지
- **재료 관리** (차별화):
  - 재료명, 재료 이미지, 칼로리, 맛 프로필
  - 메뉴별 재료 연결
- **드래그 앤 드롭 메뉴 순서 조정** (차별화)
- 이미지 파일 업로드 (서버 저장)
- 필수 필드 검증, 가격 범위 검증

## 5. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 실시간 업데이트 2초 이내
- 3D 렌더링 60fps 유지 (최적화된 모델 사용)
- 페이지 초기 로딩 3초 이내

### NFR-02: 사용성
- 터치 친화적 UI (태블릿 최적화)
- 직관적 네비게이션 (3탭 이내 목표 도달)
- 다크 모드 프리미엄 테마 일관성

### NFR-03: 호환성
- 모던 브라우저 지원 (Chrome, Safari, Firefox)
- 태블릿 해상도 최적화 (768px ~ 1024px)
- 데스크톱 관리자 화면 (1280px+)

### NFR-04: 데이터 무결성
- 주문 데이터 트랜잭션 처리
- 세션 기반 주문 격리
- 장바구니 로컬 저장 안정성

## 6. 데이터 모델 (개요)

### 핵심 엔티티
- **Store**: 매장 정보
- **Table**: 테이블 정보 (매장 소속)
- **TableSession**: 테이블 세션 (시작/종료 시각)
- **Category**: 메뉴 카테고리
- **Menu**: 메뉴 항목 (카테고리 소속)
- **Ingredient**: 재료 정보 (이미지, 칼로리, 맛 프로필)
- **MenuIngredient**: 메뉴-재료 연결 (다대다)
- **Order**: 주문 (테이블 세션 소속)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력
- **Admin**: 관리자 계정
